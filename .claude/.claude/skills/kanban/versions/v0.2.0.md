## 关键改进点

### 1. 目录内聚化（ST-011/ST-012）
- 旧格式: `tasks/TASK-NNN.json` + `reports/TASK-NNN/` + `dispatch/`
- 新格式: `tasks/TASK-NNN/task.json` + `tasks/TASK-NNN/iteration-N/` + `tasks/TASK-NNN/dispatch/`
- 自动迁移: `kanban_init_env` 时检测并迁移旧格式
- 8 个路径 helper 函数兼容新旧格式

### 2. Plan 质量门禁（ST-013/ST-014）
- 4 维评分: 需求清晰度(0.25) + 技术可行性(0.25) + 拆解合理性(0.25) + 验收标准(0.25)
- 加权总分 >= pass_threshold(默认 9.0) 通过
- 不通过自动重试(最多 max_rounds 轮)
- 超过上限进入 user_decision

### 3. Worktree 生命周期（ST-001~ST-008）
- worktree_validate: 统一校验函数
- worktree_create: 幂等创建
- worktree_merge: 容错合并（无 worktree 时兜底 commit）
- worktree_cleanup: 安全清理
- hot iteration 时自动重建 worktree

### 4. 归档自演进（ST-015）
- framework_self_assess: 读取 pitfalls + decisions
- 自动评估框架规则完整性、agent 能力覆盖
- 改进项自动通过 kanban_create_task 创建新任务

### 5. 变更全景展示
- kanban_changes_summary: 6 维结构化展示
- 需求背景 → 技术方案 → 任务拆解 → 质量评估 → 变更文件 → 可选操作
- 跨 worktree/主目录 artifact 查找

## 升级指南

### 从 v0.1.0 升级到 v0.2.0

1. **替换文件**: 将 `.claude/skills/kanban/lib/` 下的所有 .sh 文件替换为新版本
2. **自动迁移**: 首次执行 `kanban_init_env` 时自动将旧格式任务迁移到新目录结构
3. **更新 SKILL.md**: 替换为新版 SKILL.md
4. **更新 .gitignore**: 追加运行时数据和敏感文件排除规则
5. **创建版本目录**: `mkdir -p .kanban/versions/`
6. **更新 config.json**: 添加 `"version": "0.2.0"` 字段（可选）

### 破坏性变更

- `kanban_archive_task` 行为变更: 支持 new-format 目录归档
- `report_dir()` 返回值变更: 优先返回 `tasks/TASK-NNN/iteration-N/`
- 新增 `_find_artifact()` 用于跨位置查找产物
