# v0.11.0 — Issue 批量修复 + 自迭代链路 + 开发体验增强

**日期**: 2026-05-09
**关联任务**: TASK-001 ~ TASK-006（框架自修）

## Added

### Evaluator 自动评分链路 (#97)
- `evaluator collect-scores`: 自动读取 4 角色评估报告，计算 avg_score
- `evaluator record-score`: 保存 per-iteration 评分到 task.json `score_history`
- `self-improve-check`: 无参数时自动从 score_history 读取最新评分
- Task 类型新增 `scores` 和 `score_history` 字段

### 迭代产物隔离 (#97)
- `workflow start-iteration hot/full`: 增量迭代计数器，自动迁移旧 execution artifacts 到 `iteration-N/`
- hot: 重置到 execute; full: 重置到 plan

### 评分趋势 UI (#97)
- `useScoreChart.js`: 自动识别 score_history/reports 双格式，pass_threshold 9.0 参考线
- 节点颜色区分及格（绿）/不及格（红），迭代编号标注

### 轻量模式硬性门控 (#90)
- `cmd_run` 返回 `requires_confirmation: true` 当 lightweight_available
- SKILL.md 增加 AskUserQuestion 硬性门控（brainstorming + lightweight 双场景）

### Agent 调度增强 (#96)
- `workflow get-phase-agents` 返回 `scheduling` 元数据（timeout/max_parallel/poll_interval）
- 读取 config.json 的 timeout 和 scheduler 配置

### 跨任务冲突检测 (#95)
- `Guard.check_cross_task_conflicts()`: 扫描所有活跃任务检测 file_ownership 重叠
- CLI: `guard check-cross-task-conflicts`

### 知识库语义搜索 (#99 P0)
- `KnowledgeManager.search_scored()`: 结构化解析 knowledge-log.md
- 加权评分：标题×3 + 分类×2 + 内容×1，回退到行级搜索

### Agent 注册表 + 预算 (#99 P1)
- `Scheduler.scan_agents()`: 扫描 `.claude/agents/*.md` 自动发现
- `check-env` 增强: 异常检测（框架源码下的 .kanban/、多 .kanban/ 共存）

### 项目根目录自动检测
- `Filesystem.find_project_root()`: 向上查找 `.kanban/config.json`，支持 `KANBAN_ROOT` 环境变量
- 全部 14 处 `Path.cwd()` 替换为 `find_project_root()`

### NLP → LLM 推理
- `cmd_nlp` 返回 `interpret_by_llm: true` + 完整命令列表
- 删除 1000 行关键词匹配逻辑和 `nlp_patterns.json` 依赖
