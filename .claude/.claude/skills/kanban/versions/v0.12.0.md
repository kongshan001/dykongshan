# v0.12.0 — 并行调度增强 + 运行时缺陷修复 + 评估分数门控

**日期**: 2026-05-10
**关联任务**: TASK-081, TASK-082, TASK-083, TASK-096

## Added

### Plan Review 6 维度并行调度 (TASK-081)
- `Scheduler.PLAN_REVIEW_DIMENSIONS`: 6 个评审维度（requirement_clarity, technical_feasibility, task_decomposition, acceptance_criteria, research_completeness, parallel_safety）
- `Scheduler.plan_review_dimensions()`: 返回维度配置列表
- `evaluator collect-plan-review`: 收集并行维度报告，排除 applicable=false，计算加权平均
- workflow.json plan_review 阶段支持 6 个 `parallel: true` agent

### Retrospective 3 角色并行调度 (TASK-081)
- `Scheduler.RETROSPECTIVE_ROLES`: 3 个角色（retrospective_writer, acceptance_writer, knowledge_extractor）
- `Scheduler.retrospective_roles()`: 返回角色配置列表
- workflow.json retrospective 阶段支持 3 个 `parallel: true` agent

### Guard 批量并行检查 (TASK-081)
- `Guard.batch_check()`: 独立运行 4 项检查，返回 dict[str, CheckResult]
- `Guard.batch_check_combined()`: 合并所有结果为单一 CheckResult
- CLI: `guard batch-check TASK-NNN`

### Evaluate 阶段分数门槛硬性门控 (TASK-096)
- `Guard.check_evaluation_score(task)`: 读取 score_history 和 pass_threshold，调用 IterationDecider 判断
- `complete-phase` 在 evaluate 阶段自动检查分数：
  - score >= threshold → 放行
  - score < threshold 且 iteration < max → raise GuardError，消息含迭代建议（hot/full）
  - iteration >= max_iterations → 放行到 user_decision（IR-17）
- 防止编排器跳过 self-improve 直接归档

### History 自动记录 (TASK-082)
- `TaskManager.create()`: 创建时自动追加 `{"phase": "plan", "status": "started"}` 到 history
- `cmd_workflow transition`: 转换时追加 `{"phase": target, "status": "started"}` 到 history
- `cmd_workflow complete-phase`: 完成时持久化 history 到磁盘

### Version CLI 命令 (TASK-083)
- `core/cli/version.py`: 实现 version list（返回版本/Python/平台信息）和 record（记录版本文件）

## Fixed

### 新旧 Task Format 不一致 (TASK-082)
- `_write_task()` 写入新格式后自动删除旧格式文件 `tasks/TASK-NNN.json`
- 测试 helper `_read_task()` / `_set_phase()` 优先检查新格式路径
- 修复 5 个因此问题失败的 persistence 测试

### Windows 兼容性 (TASK-082)
- `scanner.py` symlink 检测改用 path-parts 匹配，兼容反斜杠路径
- 测试文件统一使用 `encoding="utf-8"` 避免 GBK 编码错误

### check-env 测试隔离 (TASK-082)
- `test_check_env` 使用 `tmp_kanban` fixture 替代 `cwd="."`

### SKILL.md Agent Type (TASK-082)
- 所有 12 处自定义 `subagent_type="kanban-*"` 替换为 `subagent_type="general-purpose"`
- Agent 行为通过 prompt 内容控制

## Changed
- `core/cli/run.py`: `cmd_workflow` transition 和 complete-phase 传入 `history=task.history`
- `core/domain/task.py`: `_write_task` 清理旧格式，`create` 追加初始 history
- `core/domain/scanner.py`: `_detect_agent_conflicts` 使用 Path.parts 匹配
- `core/infra/scheduler.py`: 新增 PLAN_REVIEW_DIMENSIONS 和 RETROSPECTIVE_ROLES
- `core/cli/evaluator.py`: 新增 `_collect_plan_review_scores` 和 `_PLAN_REVIEW_DIMENSIONS`
- SKILL.md: Plan Review 改为 6 并行维度 + 串行回退，Retrospective 改为 3 并行角色
