# v0.1.0 — Kanban 多 Agent 编排框架初始版本

**日期**: 2026-04-28
**关联任务**: TASK-001 ~ TASK-009
**迭代轮次**: 多任务迭代

## 改动摘要

Kanban 框架的第一个正式版本，建立了基于 FSM 的多 Agent 任务编排核心能力。框架从零开始构建，包含完整的工作流引擎、多角色评估体系、Worktree 隔离执行机制和自迭代循环。

## 核心特性

### FSM 工作流引擎
- 5 阶段状态机: plan → execute → evaluate → user_decision → archive
- 每个阶段有明确的入口条件、产物要求和退出条件
- 阶段转换由 Guard 保护，不允许跳过

### 多角色评估
- 4 个独立评估角色: code_reviewer / qa / pm / designer
- 10 分制统一评分，pass_threshold 可配置
- 4 角色并行评估（run_in_background）

### Guard 三层防护
1. 阶段转换检查 — 只允许合法的状态转换
2. 产物完整性检查 — 每个阶段必须产出规定文档
3. 评分标准检查 — 评估必须覆盖全部 4 个角色

### Worktree 隔离
- 每个 task 创建独立 git worktree
- 代码在隔离分支中开发
- 完成后合并到主干

### 自迭代
- Hot iteration: 评分 >= 7.0 且无架构问题，从 Execute 重来
- Full iteration: 从 Plan 重来
- 最大迭代轮次可配置（默认 6 次）

### Iron Rules 铁律
- IR-01 ~ IR-11: 11 条不可违反的框架铁律
- 涵盖 Guard、产物、评估、迭代、归档等核心约束

## Agent 定义

| Agent | 职责 |
|-------|------|
| planner | 需求分析 + 任务拆解 |
| executor | 编码实现 + 测试 |
| code-reviewer | 代码审核 |
| qa | 测试验证 |
| pm | 需求评审 |
| designer | 设计评审 |
| knowledge-manager | 知识沉淀 |
| researcher | 技术调研 |
