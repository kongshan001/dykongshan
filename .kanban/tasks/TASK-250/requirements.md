# 需求分析: 旅行实时航班追踪

## 功能需求
- FR-001: 输入航班号实时查询航班状态（起飞/到达/延误/取消）
- FR-002: 航班状态变更推送通知（延误、登机口变更、取消等）
- FR-003: 航班轨迹地图实时展示
- FR-004: 支持批量追踪多个航班
- FR-005: 历史航班数据查询与统计

## 非功能需求
- NFR-001: 航班状态更新延迟 < 60 秒
- NFR-002: 支持 300+ 航空公司数据
- NFR-003: 推送通知到达率 > 99%
- NFR-004: 地图渲染帧率 > 30fps

## 技术约束
- 航班数据源使用 AviationStack / FlightAware API
- 地图使用 Mapbox GL / Deck.gl
- 推送使用 Firebase Cloud Messaging

## 验收标准
- AC-001: 输入航班号 2 秒内返回当前状态
- AC-002: 延误/取消通知在状态变更后 60 秒内送达
- AC-003: 航班轨迹在地图上实时更新
- AC-004: 可同时追踪至少 10 个航班
