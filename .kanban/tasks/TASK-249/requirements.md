# 需求分析: 旅行同伴匹配器

## 功能需求
- FR-001: 用户填写旅行偏好问卷（目的地、预算、出行风格、语言）
- FR-002: 基于偏好算法匹配旅行同伴，推荐匹配度排序
- FR-003: 支持发起同行邀请与即时聊天
- FR-004: 用户可查看同伴历史评价和旅行记录
- FR-005: 支持创建"找同伴"公开帖，接收报名

## 非功能需求
- NFR-001: 匹配算法响应时间 < 500ms
- NFR-002: 支持多语言偏好（中/英/日）
- NFR-003: 用户隐私保护，非公开信息需授权可见
- NFR-004: 聊天消息端到端加密

## 技术约束
- 匹配算法使用向量相似度（embedding）
- 实时聊天基于 WebSocket
- 用户画像存储使用 PostgreSQL + pgvector

## 验收标准
- AC-001: 完成偏好问卷后 1 秒内返回匹配结果
- AC-002: 可发起同行邀请并收到对方回复
- AC-003: 聊天消息实时送达，延迟 < 200ms
- AC-004: 匹配结果按相似度降序排列
